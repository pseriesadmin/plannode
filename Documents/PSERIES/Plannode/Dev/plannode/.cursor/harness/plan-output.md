## P-3. 범위

**포함 (이번 아젠다 — GATE 이후 구현은 Step3·GATE B에서 세분화)**

- **접기/펼치기:** 부모 노드에 **자식이 있을 때만** 의미 있는 UI; 접기 시 해당 서브트리 **숨김**(자식 노드·해당 간선 비표시 또는 스킵). 버튼 위치: **우측분포** = **`+` 좌측**, **하위분포** = **`+` 위** — 첨부 시안과 동일한 상대 배치. 상태는 최소 **세션 단위**(재진입·저장 정책은 TASK에서 확정).
- **연결선 강조:** **현재 선택 노드** 기준 **조상·자손 체인**에 해당하는 간선만 **시각적 강조**(예: `stroke-width`·색 대비). 비선택 경로는 기존 대비 유지.
- **우측분포 겹침:** **컬럼/행 자동 배치** 경로에서 **수직 간격 부족**을 해소(레이아웃 상수·`bld`/`lm`·분포 모드 분기 — 파일럿 명세 §2와 정합).

**트리뷰 보호 (필수 한 줄):** `#CV`/`#EG` 동기 · **`drawEdges` 후 `updMM`** 계약 · **transform 컨테이너** 불변 — 변경은 **노드 카드 UI·간선 스타일·레이아웃 계산**에 한정하고 트리 **SSoT(`nodes`·`parent_id`)** 를 깨지 않는다(`AGENTS.md` **GP-13**).

**제외 (이번 plan-output에서 미리 확정하지 않음 — Step3·GATE B에서만)**

- **본 문서 단계의 실제 코드 수정·커밋** — `@promptor` 절대 금지.  
- **§3 F2-4 IA·와이어**, **§10 LLM·F2-5**, **§11 DB·`ai_generations`** — 제외.  
- **Supabase·RLS·스키마 변경** — 제외.  
- **접기 상태의 클라우드 동기 필수** — 아젠다에 없음; 필요 시 TASK 하위 옵션으로만.

**선행 결정 사항 (GATE B·TASK에서 택일)**

- **접기 상태**를 **localStorage/프로젝트 메타**에 둘지, **세션만** 둘지.  
- **강조 색·두께** 토큰 — `plannode-ui-identity.mdc`와 충돌 시 택일.  
- **우측분포**에서 겹침이 나는 **특정 트리 패턴**(깊이·자식 수) 재현 시나리오를 TASK에 명시.

**참고 파일**

- `plannode.js` — `render`, `bld`/`lm`, `drawEdges`, 분포 모드, 노드 버튼·`.nw`.

## P-4.5. PRD 연계 (`plannode-prd.mdc`, § 단위)

**IA vs LLM (1문장):** 이번 작업은 **§3 F2-4·F2-5(§10)** 가 아니라 **§3 M1 노드 트리 에디터** — 특히 **F1-1 노드 CRUD**(서브트리 표시/숨김은 트리 조작 UX) · **F1-2 캔버스**(드래그·줌·**간선·배치**) · **§4 비기능**(가독성·응답) 이다.

| PRD 앵커 | 연계 내용 |
|----------|-----------|
| **§3 M1 · F1-1** | 무한 뎁스 트리 — **접기**는 하위 가지를 일시적으로 비노출하여 **탐색 부담 완화**(데이터 삭제 아님). |
| **§3 M1 · F1-2** | 캔버스 — **패닝·줌**과 함께 **간선 가시성**(강조) · **자동 배치 가독성**(겹침 제거)이 동일 모듈 요구. |
| **§3 M1 · F1-3** | (참고) 배지 로직과 무관한 변경은 건드리지 않음 — **범위 밖 선제 금지**(GP-12). |
| **§4 비기능** | 트리 구조 **한눈에 파악** — 연결선 강조·겹침 해소가 **가독성** 목표와 정합. |
| **§1.2 USP / §3 상단** | 노드 맵 **시각 설계** — 캔버스 UX 개선이 제품 가치와 직접 연결. |

**편집·저장 귀속 (1문장):** 노드 **데이터 모델·영속화 계약**은 유지하고, **표현층(숨김·선 스타일·좌표)** 만 확장한다 — **§11·§10** 과 혼동 금지.
