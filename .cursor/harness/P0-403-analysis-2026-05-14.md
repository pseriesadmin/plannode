### 원인 2️⃣: 삭제된 프로젝트 로컬 캐시 정리 안 됨

**`syncProjectsForModalList()` 흐름 (`+page.svelte` 라인 500~563):**

```typescript
async function syncProjectsForModalList() {
  // 1단계: 클라우드에서 프로젝트 목록 가져오기
  const cloudRes = await fetchOwnWorkspaceProjectMetasForModal();
  mergedList = cloudRes.ok ? mergeModalListCloudCanon(...) : [...plist];

  // 2단계: 공유 슬라이스 필터
  const myAclProjectIds = new Set(rows.map((r) => r.project_id));

  // 3단계: 각 프로젝트 접근 권한 확인
```

**문제점:**
- ✅ `canAccessProject()` 함수는 **RLS 기반 ACL 행 검사**를 함
- ✅ 소유자 삭제 후 공유자 ACL 행은 `plan_project_acl` 테이블에서 삭제됨
- ❌ **하지만 로컬 `projects` 스토어는 즉시 갱신되지 않음**
- 🔴 `canAccessProject()` 호출 시 이미 삭제된 프로젝트를 Supabase에서 다시 쿼리 → 네트워크 지연 + **RLS 거부 but 스토어는 여전히 메모리에 있음**
