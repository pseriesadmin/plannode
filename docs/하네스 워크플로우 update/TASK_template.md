## 피그마·UI 원칙 (화면·레이아웃·컴포넌트 작업이 하나라도 있으면 필독)

1. **피그마 URL만으로 구현을 시작하지 않는다.** (링크는 SPA·캡처 한계로 요소·좌표가 고정되지 않음.)
2. **요소·역할·DS**를 문서에 남긴다.
   - **요소**: 라벨·문구·섹션
   - **역할**: 체크박스 / 주 버튼 / 보조 버튼 / 링크 / 입력 / …
   - **DS**: `.cursor/rules/ui-ux-quick-reference.md` 기준 **토큰·컴포넌트 이름** (미확정이면 `DS 미정 → Stephen 확인`)
3. 아래 **UI 스펙 표** 또는 각 NOW의 **`| UI:`** 줄이 비어 있으면 step4에서 **질문·BLOCKED**될 수 있다.

## 맥락·에스컬레이션 (step4 실행 중)

**전제**: `TASK.md`는 Cursor가 해석하는 YAML 런타임이 **아니다**. 아래는 **자동 분기가 아니라** Stephen·에이전트가 따를 **운영 플레이북**이다. (모델 이름은 Cursor에 맞게 바꿔 쓴다.)

| 트리거 | 권장 행동 |
|--------|-----------|
| **에러 연속·타입/빌드 실패** | 채팅 모델을 **요약·정리에 유리한 경량 모델**(예: composer-2-fast 등)로 전환 → 지금까지 **변경 파일·실패 원인 5줄 요약** 후 같은 NOW로 `@gsd-agent` 재개 |
| **DB/API 스키마 변경** | 구현 중단 → `schema-change-history.mdc`·관련 API 확인 → (선택) `.agents/skills/supabase-postgres-best-practices` `@` → **GATE B·TASK·Plan 재검토** (필요 시 step3 재실행) |
| **신규 패키지·외부 의존** | **BACKLOG**에 분해 태스크 추가 → **GATE B 재승인** 뒤 진행 |
| **맥락 드리프트 의심** (장시간·다회 호출) | **GATE C 3회마다** 또는 **NOW 2개 완료마다** `GSD_LOG.md`에 **한 줄 스냅샷** (날짜·NOW id·한 줄 요약) |

**NOW 블록에만 예외를 두고 싶을 때** (선택):

```
  | 맥락: (선택) 이 NOW 전용 에스컬레이션 — 비우면 위 표 공통 적용
```

## NOW (현재 실행 중)

> `@gsd-agent`가 선정. **한 번에 1개만.**

```
- [ ] NEXT-1: (태스크 제목)
  | 목표: …
  | UI: (화면·컴포넌트 작업이면 필수) 영역별 요소·역할·DS — 또는 위 UI 스펙 표 # 참조. 비화면이면 생략 가능.
  | 권한: (계정·역할·참여·계약·내정보 영향 시 필수) 헌법 준수 요약 1문장 또는 `해당 없음`
  | 대상: …
  | 산출: …
  | 회귀: …
  | 예상: NN분
```
