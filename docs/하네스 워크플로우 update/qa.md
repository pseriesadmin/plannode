## 🔍 검수 1단계: 규칙 정합성 (Rules Compliance)

> 자동 코드 검색으로 확인 가능한 항목. 실행 후 결과 명시.

### 공통 보안·품질 규칙

```
□ 서버 전용 키 클라이언트 노출 없음
  → 검색: PUBLIC_KEY / public import에서 SECRET 포함 여부
□ SQL Injection 가능 코드 없음 (파라미터화 사용)
□ XSS 취약점 없음 ({@html} 또는 dangerouslySetInnerHTML 최소화)
□ 사용자 입력 검증 코드 존재
□ RLS 정책 우회 코드 없음 (SERVICE_ROLE 클라이언트 최소화)
```

### 프로젝트 도메인 규칙 (아래 중 해당 프로젝트 섹션 적용)

#### [크레이지샷] rental.mdc 준수

```
□ RULE-0-1: reservations 직접 INSERT 없음
  → 검색: supabase.from('reservations').insert 코드 없음
□ RULE-0-2: 가용성 체크 + INSERT 단일 RPC 내 처리
□ RULE-0-3: expires_at 필터 포함된 가용성 쿼리
□ RULE-0-4: TIMESTAMPTZ 사용 (DATE 단독 없음)
□ RULE-0-5: ts_range / rental_range 분리 사용
```

#### [크레이지샷] payment.mdc 준수

```
□ RULE-P-0-1: 결제창 오픈 전 atomic_reserve_asset 호출
□ RULE-P-0-2: idempotency_key 모든 결제 요청 포함
□ RULE-P-0-3: 실결제 ↔ 보증금 분리 (합산 코드 없음)
□ RULE-P-0-4: 웹훅 멱등성 처리 (중복 수신 체크)
```

```
□ FO-0-5: B2G 계약서 번호 자동 채번 RPC 사용
```

#### [공통] ui-mobile.mdc 준수

```
□ 인터랙티브 요소 44×44px 이상
```

## 🔍 검수 2단계: 기술 부채 점검 (Technical Debt)

```
코드 품질
□ console.log 잔류 0건 (grep 확인)
□ TODO / FIXME 주석 0건 또는 TASK.md BACKLOG 등록됨
□ any 타입 사용 0건 (TypeScript strict 기준)
□ 미사용 import 없음
□ 에러 처리 누락 없음 (try-catch / error() 헬퍼)
□ 함수·변수명 명확성 (축약어 최소화)

성능
□ N+1 쿼리 없음 (join 활용 확인)
□ 대형 이미지 직접 참조 없음 (CDN URL 사용)
□ 불필요한 실시간 구독 없음 (onDestroy 해제 확인)
□ 불필요한 전역 상태 없음

접근성 (기본)
□ 이미지 alt 속성 존재
□ 버튼/링크 명확한 레이블
□ 색상만으로 정보 전달하지 않음
```

## 🔍 검수 3단계: 회귀 테스트 확인 (v2 신규)

> 이번 구현이 기존 기능을 깨뜨리지 않았는지 확인.

```
□ npm run test (또는 프로젝트 테스트 명령) 전체 실행
  → 이번 사이클 이전 테스트도 모두 통과하는가?
  → 실패한 테스트가 있다면 이번 변경으로 인한 것인가?

□ TypeScript 컴파일 체크
  → npm run check (또는 tsc --noEmit)
  → 에러 0건 확인

□ 기존 핵심 경로 영향도 확인
  → 이번 변경 파일과 import 관계 있는 파일 목록 제시
  → Stephen이 추가 확인할 파일 안내
```
